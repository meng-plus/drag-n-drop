drop-n-drop配置说明
1. 配置文件系统
```c
#define VFS_DISK_SIZE (MB(64))
```
注意上面的只影响展示大小，影响实际写入大小的是在virtual_fs.c文件中file_allocation_table_t结构体大小需要根据实际情况进行调整
例如8MB的文件系统，需要调整为：
```c
typedef struct file_allocation_table
{
    uint8_t f[512 * 8];
} file_allocation_table_t;
```
2. 新增文件
从vfs_user_build_filesystem函数中新增文件，例如：
```c
vfs_create_file("DETAILS TXT", read_file_details_txt, 0, file_size);
```
注意：文件名需要符合8.3格式
文件名只允许使用大写字母、数字和下划线，并且不能超过8个字符 文件名不区分大小写
文件名与类型 扩展名 之间必须使用一个空格分隔
例如：DETAILS TXT
不足8个字符的文件名，需要补齐空格，例如：DETAILS TXT
超过8个字符的文件名，需要截取前8个字符，例如：DETAILS TXT

uint32_t read_file_details_txt(uint32_t sector_offset, uint8_t *data, uint32_t num_sectors)
{

}
```

```
3. 处理文件变更
从vfs_user_file_change_handler函数中处理文件变更，例如：
```c
vfs_user_file_change_handler(const vfs_filename_t filename, vfs_file_change_t change, vfs_file_t file, vfs_file_t new_file_data);
```
4. 周期性调用
从vfs_mngr_periodic函数中周期性调用，例如：
```c
vfs_mngr_periodic(elapsed_ms);
```
